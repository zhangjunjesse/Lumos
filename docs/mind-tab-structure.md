# Tab 结构重新设计

## 问题分析

### 当前问题
- 8个Tab太多：aboutYou, aboutLumos, howWeWork, memories, persona, rules, intelligence, experience
- 统计信息（experience）无意义
- 技术配置（intelligence）不应暴露
- 功能重复：persona/rules 应该整合到 aboutLumos

### 用户困惑
- 不知道该点哪个Tab
- 记忆、人格、规则的关系不清晰
- 信息过于分散

---

## 新Tab结构（3个核心Tab）

### Tab 1: 关于你 ⭐️
**英文：** About You
**定位：** Lumos眼中的主人画像（核心模块）

**包含内容：**
- 基本信息（职业、兴趣、习惯）
- 工作风格（技术栈、工作流程）
- 沟通偏好（回复风格、详细程度）
- 当前状态（正在做的项目、近期目标）

**数据来源：**
- 对话中自动提取的记忆（category='user_profile'）
- 用户手动补充的信息

**展示方式：**
- 卡片式布局，按类别分组
- 每张卡片显示：内容、来源、更新时间
- 显示"了解程度"进度条

---

### Tab 2: 关于Lumos
**英文：** About Lumos
**定位：** Lumos的人格、能力和行为规则

**包含内容：**

#### 2.1 人格特质（Persona）
- 回复风格（友好/专业/幽默程度）
- 语言风格（简洁/详细、正式/随意）
- 专业领域（擅长的技术栈）

**数据来源：** persona表

#### 2.2 行为规则（Rules）
- 工作原则（如何处理任务）
- 禁忌事项（不该做什么）
- 优先级（什么最重要）

**数据来源：** rules表

**展示方式：**
- 上半部分：人格特质（预设模板 + 自定义）
- 下半部分：行为规则（列表形式，可添加/编辑）

---

### Tab 3: 相处之道
**英文：** How We Work
**定位：** 协作偏好和互动规则

**包含内容：**
- 工作模式（自动批准 vs 需要确认）
- 通知偏好（何时提醒、如何提醒）
- 隐私边界（可访问的数据范围）
- 协作习惯（主动建议 vs 等待指令）

**数据来源：**
- 系统设置
- 用户偏好配置

**展示方式：**
- 开关和选项配置
- 场景化说明

---

## 数据整合方案

### 原8个Tab → 新3个Tab映射

| 原Tab | 去向 | 说明 |
|-------|------|------|
| aboutYou | **Tab 1: 关于你** | 保留，作为核心Tab |
| memories | **Tab 1: 关于你** | 整合进去，作为数据来源 |
| aboutLumos | **Tab 2: 关于Lumos** | 保留框架 |
| persona | **Tab 2: 关于Lumos** | 整合为"人格特质"部分 |
| rules | **Tab 2: 关于Lumos** | 整合为"行为规则"部分 |
| howWeWork | **Tab 3: 相处之道** | 保留 |
| intelligence | **删除** | 技术配置不暴露给用户 |
| experience | **删除** | 统计信息无意义 |

## 实施方案

### 1. Tab组件更新
```typescript
const tabs = [
  { id: 'aboutYou', label: t('mind.aboutYou'), icon: User },
  { id: 'aboutLumos', label: t('mind.aboutLumos'), icon: Brain },
  { id: 'howWeWork', label: t('mind.howWeWork'), icon: Settings }
];
```

### 2. 数据查询逻辑

**Tab 1 - 关于你：**
```sql
SELECT * FROM memories
WHERE category = 'user_profile'
AND isArchived = 0
ORDER BY isPinned DESC, updatedAt DESC
```

**Tab 2 - 关于Lumos：**
```sql
-- 人格特质
SELECT * FROM persona WHERE scope = 'global'

-- 行为规则
SELECT * FROM rules WHERE isEnabled = 1
```

**Tab 3 - 相处之道：**
- 从系统设置读取
- 不需要数据库查询

## 总结

### 简化效果
- 8个Tab → 3个Tab
- 删除2个无用Tab（intelligence, experience）
- 整合3个重复Tab（memories, persona, rules）

### 信息架构
- **关于你**：主人画像（核心）
- **关于Lumos**：AI人格+规则
- **相处之道**：协作偏好

### 用户体验
- 结构清晰，一目了然
- 功能不重复，逻辑连贯
- 符合"了解彼此"的核心理念
