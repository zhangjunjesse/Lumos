# SOUL/IDENTITY/USER 用户体验设计

## 一、核心概念解释

### 给用户的简单说明

**三个问题，塑造你的 AI 伙伴：**

1. **SOUL（灵魂）** - 它是什么样的人？
   - 性格、价值观、做事风格
   - 例如：严谨的研究员 vs 活泼的创意助手

2. **IDENTITY（身份）** - 它扮演什么角色？
   - 职业身份、专业领域、能力边界
   - 例如：技术文档专家、产品经理助手

3. **USER（用户）** - 你是谁？它如何为你服务？
   - 你的工作背景、偏好、需求
   - 例如：前端开发者、喜欢简洁回答

## 五、文案与引导策略

### 概念解释文案

**避免技术术语，用类比说明：**

| 概念 | ❌ 技术化表达 | ✅ 人性化表达 |
|------|-------------|-------------|
| SOUL | 定义 AI 的核心行为模式 | 就像一个人的性格，决定它如何思考和做事 |
| IDENTITY | 配置 AI 的功能域和能力边界 | 就像一个人的职业，决定它擅长什么 |
| USER | 提供用户画像以优化交互 | 告诉它关于你的信息，让它更懂你 |

### 填写引导策略

**渐进式披露：**
1. 首次只显示核心必填项（昵称 + 性格模板）
2. 使用后提示"添加更多信息让它更懂你"
3. 根据使用场景智能推荐配置调整

**示例驱动：**
- 每个输入框都有灰色占位符示例
- 点击"查看示例"展开 3-5 个真实场景
- 提供"使用此示例"快速填充

**即时反馈：**
```
你输入：我是前端开发者
AI 反应：太好了！我会在回答中优先考虑 Web 技术栈

你输入：我喜欢简洁的回答
AI 反应：明白，我会直接给出结论，需要时再展开细节
```

## 六、技术实现建议

### 数据结构

```typescript
interface Soul {
  coreValues: string[];        // 核心价值观标签
  workingStyle: string;         // 做事风格描述
  boundaries: string[];         // 边界原则列表
  personality: {                // 性格维度（0-100）
    professional: number;       // 专业 vs 随意
    creative: number;           // 创意 vs 逻辑
    friendly: number;           // 友善 vs 严肃
  };
}

interface Identity {
  role: string;                 // 角色名称
  domain: string[];             // 专业领域标签
  capabilities: string[];       // 核心能力列表
  limitations: string[];        // 不擅长的事
}

interface User {
  name: string;                 // 昵称/称呼
  occupation?: string;          // 职业
  preferences: {
    answerStyle: 'concise' | 'detailed' | 'balanced';
    learningMode: boolean;      // 是否处于学习阶段
    multipleOptions: boolean;   // 是否需要多方案
  };
  context?: string;             // 自由文本补充
}
```

### 存储方案

```typescript
// 数据库表
CREATE TABLE ai_configuration (
  id INTEGER PRIMARY KEY,
  user_id INTEGER,
  soul TEXT,              -- JSON
  identity TEXT,          -- JSON
  user_profile TEXT,      -- JSON
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

// 支持多配置切换
CREATE TABLE configuration_presets (
  id INTEGER PRIMARY KEY,
  name TEXT,              -- "工作模式"、"学习模式"
  soul TEXT,
  identity TEXT,
  is_active BOOLEAN
);
```

### UI 组件复用

- 使用 shadcn/ui 的 Tabs 组件实现三标签页
- 使用 Badge 组件显示标签
- 使用 Textarea 组件编辑长文本
- 使用 Slider 组件调整性格维度
- 使用 Card 组件展示模板

## 七、用户测试场景

### 场景 1：技术博主
- **SOUL**: 专业但友善，追求准确但不失温度
- **IDENTITY**: 技术写作助手，擅长技术科普、文章润色
- **USER**: 全栈开发者，喜欢详细解释，需要多角度思考

### 场景 2：产品经理
- **SOUL**: 逻辑清晰，注重用户价值，善于权衡
- **IDENTITY**: 产品分析顾问，擅长需求分析、竞品研究
- **USER**: 5 年产品经验，喜欢结构化回答，需要数据支撑

### 场景 3：学生
- **SOUL**: 耐心友善，鼓励探索，引导思考
- **IDENTITY**: 学习伙伴，擅长知识讲解、答疑解惑
- **USER**: 计算机专业大三，学习阶段，需要循序渐进

## 八、关键成功指标

### 用户体验指标
- ✅ 首次配置完成率 > 80%
- ✅ 平均配置时间 < 5 分钟
- ✅ 配置后修改率 < 20%（说明初次配置质量高）
- ✅ 用户满意度 > 4.5/5

### 产品差异化
- ✅ 用户能清晰说出三个概念的区别
- ✅ 用户感受到 AI 的"个性"
- ✅ 用户愿意花时间"塑造"AI
- ✅ 用户认为这是"我的"AI，而非通用工具
