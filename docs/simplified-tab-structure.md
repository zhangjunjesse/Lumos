# 简化 Tab 结构方案

## 问题分析

**当前问题：**
- 8 个 Tab 太多，用户找不到重点
- 统计、配置等技术细节不应该暴露
- 人格、规则、记忆的边界不清晰

**设计原则：**
- 用户视角，不是技术视角
- 核心功能优先，技术细节隐藏
- 清晰的信息架构

---

## 最终方案：3 个 Tab

```
┌─────────────────────────────────────┐
│ [关于你] [Lumos的性格] [相处方式]    │
└─────────────────────────────────────┘
```

### 为什么是 3 个？

1. **关于你** - 主人画像（核心）
2. **Lumos的性格** - AI 如何表现
3. **相处方式** - 互动规则

简单、清晰、符合用户心智模型。

## 删除的内容

### 不再需要的 Tab
- ❌ 统计 Tab - 技术细节，用户不关心
- ❌ 配置 Tab - 分散到各 Tab 内
- ❌ 高级设置 Tab - 简化为基础选项
- ❌ 日志 Tab - 技术调试用，不展示
- ❌ 导入导出 Tab - 放到设置菜单

### 简化的功能
- ❌ 复杂的性格雷达图 → 简单的单选/多选
- ❌ 详细的统计数据 → 只保留"了解程度"进度条
- ❌ 原始 JSON 编辑 → 只提供表单编辑
- ❌ 技术参数调整 → 使用合理默认值

## 实施建议

### 数据映射

**关于你 Tab：**
```typescript
{
  basic: { name, occupation, currentProject },
  workStyle: { pace, problemSolving },
  expertise: { skills: string[] },
  confidence: number // 了解程度
}
```

**Lumos的性格 Tab：**
```typescript
{
  tone: 'professional' | 'friendly' | 'casual',
  answerStyle: ['concise', 'detailed', 'multiple-options'],
  language: 'formal' | 'casual'
}
```

**相处方式 Tab：**
```typescript
{
  proactiveness: 'reactive' | 'suggestive' | 'proactive',
  boundaries: ['admit-uncertainty', 'acknowledge-limits'],
  memory: {
    enabled: boolean,
    rememberHistory: boolean,
    rememberPreferences: boolean
  }
}
```

### UI 组件

- Radio Group（单选）
- Checkbox Group（多选）
- Input（文本输入）
- Tag Input（标签输入）
- Progress Bar（进度条）
- Button（编辑、清除）

## 对比：简化前 vs 简化后

| 维度 | 简化前 | 简化后 |
|------|--------|--------|
| Tab 数量 | 8 个 | 3 个 |
| 技术细节 | 暴露统计、配置 | 完全隐藏 |
| 信息层级 | 扁平、混乱 | 清晰、分层 |
| 用户认知 | 需要理解技术概念 | 符合日常认知 |
| 操作复杂度 | 需要多处配置 | 集中在 3 个页面 |

## 用户体验流程

### 首次使用
1. 进入"关于你" → 填写基本信息
2. 进入"Lumos的性格" → 选择性格基调
3. 进入"相处方式" → 设置互动规则
4. 完成，开始使用

### 日常调整
- 想改 AI 回答风格 → "Lumos的性格"
- 想更新个人信息 → "关于你"
- 想调整主动性 → "相处方式"

清晰、直观、易找。
