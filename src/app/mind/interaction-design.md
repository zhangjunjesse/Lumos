# Mind 模块交互设计

## 设计目标

让用户感受到在"培养"一个 AI 伙伴，而非配置一个工具。

**核心理念：**
- 首次使用：从陌生到熟悉的"相识"过程
- 日常编辑：轻松有趣的"培养"体验
- 长期使用：见证 AI 伙伴的"成长"

---

## 一、首次使用引导流程（Onboarding）

### 1.1 引导策略

**目标：** 3 分钟内让用户理解价值并完成初始设置

**原则：**
- 不要一次性展示所有功能（避免信息过载）
- 让用户快速看到效果（即时反馈）
- 提供跳过选项（尊重高级用户）

### 1.2 引导步骤

#### Step 1: 欢迎界面（5 秒）
```
┌─────────────────────────────────────┐
│                                     │
│         👋 认识一下，我是 Lumos      │
│                                     │
│    一个可以根据你的需求调整性格的    │
│           AI 助手                   │
│                                     │
│    [开始设置] [稍后再说]             │
│                                     │
└─────────────────────────────────────┘
```

**文案要点：**
- 用第一人称（"我是"而非"这是"）
- 强调"可调整"（给用户控制感）
- 提供跳过选项

#### Step 2: 快速问答（30 秒）
```
┌─────────────────────────────────────┐
│ 让我了解一下你的偏好                 │
│                                     │
│ 1. 你希望我的回答风格是？            │
│    ○ 简洁直接  ○ 详细解释           │
│    ○ 幽默轻松  ○ 专业严谨           │
│                                     │
│ 2. 你主要用我做什么？                │
│    ☑ 写代码    ☑ 写文档             │
│    ☐ 学习      ☐ 创意               │
│                                     │
│ [下一步]                            │
└─────────────────────────────────────┘
```

**设计要点：**
- 单选 + 多选混合（降低决策负担）
- 问题不超过 3 个
- 每个问题都有默认选项

#### Step 3: 即时预览（30 秒）
```
┌─────────────────────────────────────┐
│ 根据你的选择，我会这样回答：         │
│                                     │
│ ┌─────────────────────────────┐   │
│ │ 用户：如何优化这段代码？     │   │
│ │                             │   │
│ │ Lumos：这段代码可以用...    │   │
│ │ [简洁版本示例]              │   │
│ └─────────────────────────────┘   │
│                                     │
│ 满意吗？                            │
│ [很好] [再调整一下]                 │
└─────────────────────────────────────┘
```

**设计要点：**
- 用真实对话示例展示效果
- 提供"再调整"选项（降低决策压力）
- 强调"可以随时修改"

#### Step 4: 完成引导（5 秒）
```
┌─────────────────────────────────────┐
│         ✨ 设置完成！                │
│                                     │
│    你可以随时在"内在世界"调整我的    │
│         性格和知识                  │
│                                     │
│    [开始对话] [先看看设置]           │
└─────────────────────────────────────┘
```

## 二、日常编辑流程

### 2.1 编辑入口

**场景 1：从对话触发**
```
用户：你能不能回答得更简洁一点？
Lumos：好的！我已经调整了回答风格。
      [查看调整] [撤销]
```

**场景 2：主动编辑**
```
设置页面 → "内在世界" Tab → 编辑按钮
```

### 2.2 编辑界面设计

#### 布局：三栏式
```
┌──────────┬──────────────────┬──────────┐
│          │                  │          │
│  导航    │   编辑区         │  预览    │
│          │                  │          │
│ SOUL     │  [编辑内容]      │  效果    │
│ IDENTITY │                  │  展示    │
│ USER     │                  │          │
│          │                  │          │
└──────────┴──────────────────┴──────────┘
```

**设计要点：**
- 左侧导航：快速切换三个部分
- 中间编辑：提供模板和 AI 建议
- 右侧预览：实时展示效果

### 2.3 让编辑变得轻松

#### 功能 1：智能模板
```
┌─────────────────────────────────────┐
│ SOUL - 核心性格                     │
│                                     │
│ [从模板开始]                        │
│  • 专业助手（适合工作场景）          │
│  • 学习伙伴（适合学习场景）          │
│  • 创意搭档（适合头脑风暴）          │
│  • 自定义                           │
└─────────────────────────────────────┘
```

#### 功能 2：AI 辅助编写
```
┌─────────────────────────────────────┐
│ 你希望我：                          │
│ [输入框]                            │
│                                     │
│ 💡 AI 建议：                        │
│ "根据你的描述，我建议这样定义..."   │
│ [采用] [修改] [忽略]                │
└─────────────────────────────────────┘
```

#### 功能 3：快捷操作
```
- Cmd+K：AI 帮我完善
- Cmd+E：查看示例
- Cmd+S：保存并预览
```

### 2.4 让编辑变得有趣

#### 游戏化元素 1：成长进度
```
┌─────────────────────────────────────┐
│ Lumos 的成长                        │
│                                     │
│ ████████░░ 80% 完善度               │
│                                     │
│ ✓ 核心性格已定义                    │
│ ✓ 知识领域已设置                    │
│ ⚠ 用户偏好还可以更详细               │
│                                     │
│ [继续完善]                          │
└─────────────────────────────────────┘
```

#### 游戏化元素 2：对话次数里程碑
```
┌─────────────────────────────────────┐
│ 🎉 你们已经对话 100 次了！           │
│                                     │
│ Lumos 现在更了解你的习惯了。         │
│ 要不要调整一下我的回答方式？         │
│                                     │
│ [去看看] [下次再说]                 │
└─────────────────────────────────────┘
```

#### 游戏化元素 3：个性化徽章
```
解锁成就：
- 🎯 "精准定义者"：完善度达到 90%
- 🔧 "调教大师"：修改超过 10 次
- 💬 "老朋友"：对话超过 500 次
```

## 三、关键交互细节

### 3.1 按钮设计

**主要操作：**
- 保存：`[保存更改]` - 蓝色，右上角
- 预览：`[查看效果]` - 次要按钮，保存旁边
- 重置：`[恢复默认]` - 文字链接，底部

**次要操作：**
- AI 建议：`💡 让 AI 帮我` - 浮动按钮
- 导入/导出：`[导入配置]` - 设置菜单内

### 3.2 提示设计

**保存成功：**
```
✓ 已保存！Lumos 会在下次对话中应用新设置
```

**未保存提示：**
```
⚠ 有未保存的更改，确定要离开吗？
[保存并离开] [放弃更改] [取消]
```

**AI 建议：**
```
💡 提示：这个描述可能太抽象了，试试加一些具体例子？
```

### 3.3 动画设计

**保存动画：**
- 按钮变为 `✓ 已保存`（1 秒）
- 预览区淡入新内容（0.3 秒）

**切换动画：**
- 左侧导航高亮滑动（0.2 秒）
- 编辑区内容淡入淡出（0.3 秒）

**AI 建议出现：**
- 从底部滑入（0.3 秒）
- 轻微弹跳效果

## 四、对话式编辑（创新点）

### 4.1 场景：用户不知道怎么写

**传统方式：**
```
[空白输入框] ← 用户不知道写什么
```

**对话式方式：**
```
┌─────────────────────────────────────┐
│ 💬 不知道怎么写？我来帮你！          │
│                                     │
│ Lumos：你希望我在回答时更注重什么？  │
│                                     │
│ 用户：[输入] 我希望你更专业一点      │
│                                     │
│ Lumos：明白了！我会：                │
│ • 使用更正式的语言                  │
│ • 提供更多技术细节                  │
│ • 减少口语化表达                    │
│                                     │
│ 这样可以吗？ [可以] [再调整]         │
└─────────────────────────────────────┘
```

### 4.2 场景：用户想要快速调整

**快捷对话：**
```
用户在聊天中：@lumos 你能不能幽默一点？

Lumos：
好的！我已经调整了回答风格 😄
[查看更改] [撤销]
```

## 八、用户测试计划

### 测试目标
1. 首次引导是否清晰？
2. 编辑流程是否流畅？
3. 游戏化元素是否有趣？

### 测试方法
- A/B 测试：对比有/无引导的完成率
- 用户访谈：收集定性反馈
- 数据分析：跟踪编辑频率、完善度

---

## 九、设计原则总结

1. **渐进式披露**：不要一次性展示所有功能
2. **即时反馈**：每次操作都要有明确的反馈
3. **降低门槛**：提供模板和 AI 建议
4. **增加乐趣**：游戏化元素让编辑不枯燥
5. **尊重用户**：提供跳过和撤销选项
6. **建立情感**：用"成长"概念增强连接感
