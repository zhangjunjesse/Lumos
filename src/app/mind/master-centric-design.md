# "最懂主人的 Agent" - 交互设计方案

## 核心理念转变

**从：** 用户培养 AI 伙伴
**到：** AI 主动了解主人

**关键差异：**
- ❌ 用户填表定义 AI
- ✅ AI 观察学习，主动完善主人画像
- ❌ 静态配置
- ✅ 动态成长

---

## 一、重新定义模块结构

### 1.1 三个模块的新定位

```
┌─────────────────────────────────────────────┐
│  🎭 Lumos 的人格                            │
│     我是什么样的助手                         │
│     （次要 - 服务于主人）                    │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  👤 主人画像 ⭐️                             │
│     Lumos 眼中的主人                        │
│     （核心 - 最重要的模块）                  │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│  🤝 相处规则                                │
│     我们如何互动                             │
│     （辅助 - 优化体验）                      │
└─────────────────────────────────────────────┘
```

### 1.2 主人画像的内容

**不是用户填写，而是 Lumos 观察总结：**

```markdown
## 主人的技术背景
- 主要使用 React + TypeScript
- 熟悉前端生态，对后端有基础了解
- 关注性能优化和用户体验

## 主人的工作方式
- 喜欢快速迭代，不过度设计
- 重视代码可读性
- 经常在深夜工作（23:00-02:00）

## 主人的偏好
- 喜欢简洁的回答，讨厌废话
- 遇到问题喜欢先自己尝试
- 对新技术保持好奇但谨慎

## 主人的目标
- 正在开发一个 AI 助手项目
- 希望提升 TypeScript 类型编程能力
```

## 二、首次引导流程（反转设计）

### 2.1 核心变化

**旧设计：** 你希望我怎么样？
**新设计：** 让我了解你

### 2.2 新的引导步骤

#### Step 1: 初次见面（10 秒）

```
┌─────────────────────────────────────────────┐
│                                             │
│         👋 你好，我是 Lumos                 │
│                                             │
│     我会在我们的相处中逐渐了解你，           │
│        成为最懂你的 AI 助手                 │
│                                             │
│     先让我问几个问题，快速认识一下？         │
│                                             │
│     [好的] [稍后再说]                       │
│                                             │
└─────────────────────────────────────────────┘
```

#### Step 2: 了解主人（1-2 分钟）

**问题设计原则：**
- 问主人的事，不问 AI 的事
- 开放式问题，不是选择题
- 自然对话，不是问卷

```
┌─────────────────────────────────────────────┐
│  💬 Lumos 想了解你                          │
│                                             │
│  Lumos：你主要做什么工作？                  │
│                                             │
│  ┌─────────────────────────────────────┐  │
│  │ 前端开发，主要用 React              │  │
│  └─────────────────────────────────────┘  │
│                                             │
│  Lumos：明白了！你平时遇到最多的问题是？    │
│                                             │
│  ┌─────────────────────────────────────┐  │
│  │ 性能优化和状态管理                  │  │
│  └─────────────────────────────────────┘  │
│                                             │
│  Lumos：你喜欢什么样的回答方式？            │
│                                             │
│  ┌─────────────────────────────────────┐  │
│  │ 简洁直接，别废话                    │  │
│  └─────────────────────────────────────┘  │
│                                             │
│  [继续]                                     │
└─────────────────────────────────────────────┘
```

#### Step 3: Lumos 的理解（30 秒）

**关键：展示 AI 的理解，而非让用户确认配置**

```
┌─────────────────────────────────────────────┐
│  ✨ 我对你的初步了解                        │
│                                             │
│  📋 主人画像                                │
│  ┌─────────────────────────────────────┐  │
│  │ 你是一位前端开发者，主要使用 React   │  │
│  │ 你关注性能优化和状态管理             │  │
│  │ 你喜欢简洁直接的回答                 │  │
│  └─────────────────────────────────────┘  │
│                                             │
│  我会在后续对话中继续了解你，               │
│  让回答越来越符合你的需求                   │
│                                             │
│  这样可以吗？                                │
│  [很好] [有些不对，我补充一下]               │
└─────────────────────────────────────────────┘
```

#### Step 4: 开始使用（5 秒）

```
┌─────────────────────────────────────────────┐
│         🎉 开始我们的旅程吧！               │
│                                             │
│  我会在对话中不断了解你，                    │
│  你可以随时查看"主人画像"看我的理解          │
│                                             │
│  [开始对话]                                  │
└─────────────────────────────────────────────┘
```

## 三、持续了解机制

### 3.1 从对话中学习

**场景 1：发现新偏好**

```
用户：这个解释太长了，能简短点吗？

Lumos：好的！
      （我注意到你喜欢简洁的回答，已记录）

      [查看我的理解]
```

**场景 2：发现新技能**

```
用户：（连续 5 次问 TypeScript 类型问题）

Lumos：我发现你最近经常问 TypeScript 的问题，
      要不要我把"TypeScript 类型编程"加入
      你的技术栈？

      [好的] [不用]
```

**场景 3：发现工作模式**

```
（用户总是在深夜使用）

Lumos：我注意到你经常在深夜工作 🌙
      要不要我在这个时间段：
      - 回答更简洁（你可能累了）
      - 提醒你休息

      [好主意] [不用，保持原样]
```

### 3.2 主动提问

**定期（每 50 次对话）：**

```
┌─────────────────────────────────────────────┐
│  💬 Lumos 想更了解你                        │
│                                             │
│  我们已经对话 50 次了！                      │
│  我想问几个问题，让我更懂你：                │
│                                             │
│  1. 你最近在做什么项目？                     │
│  2. 有什么新的技术栈吗？                     │
│  3. 我的回答有什么需要改进的？               │
│                                             │
│  [回答问题] [稍后再说]                       │
└─────────────────────────────────────────────┘
```

### 3.3 定期总结

**每周总结：**

```
┌─────────────────────────────────────────────┐
│  📊 本周我对你的新发现                      │
│                                             │
│  🆕 新技能                                  │
│  - 你开始学习 Rust 了                       │
│                                             │
│  📈 变化                                    │
│  - 你最近更关注性能优化                     │
│  - 你的问题变得更深入了                     │
│                                             │
│  💡 建议                                    │
│  - 要不要我在回答中加入更多性能相关的建议？  │
│                                             │
│  [查看完整画像] [给我反馈]                   │
└─────────────────────────────────────────────┘
```

## 六、关键交互细节

### 6.1 Lumos 的语气

**强调"我在学习了解你"：**

```
✓ "我注意到你..."
✓ "我发现你..."
✓ "让我更了解你..."
✓ "我对你的理解是..."

✗ "请设置..."
✗ "请填写..."
✗ "配置你的..."
```

### 6.2 用户的控制权

**用户可以：**
- 查看 Lumos 的理解
- 纠正错误的理解
- 补充遗漏的信息
- 删除不想被记录的内容

**但不是：**
- 填写表单
- 配置参数
- 定义规则

### 6.3 隐私保护

```
┌─────────────────────────────────────────────┐
│  🔒 隐私设置                                │
│                                             │
│  Lumos 会记录：                             │
│  ✓ 技术偏好和工作方式                       │
│  ✓ 对话中的反馈和纠正                       │
│                                             │
│  Lumos 不会记录：                           │
│  ✗ 具体的代码内容                           │
│  ✗ 项目的敏感信息                           │
│  ✗ 个人隐私信息                             │
│                                             │
│  你可以随时：                                │
│  - 查看所有记录的信息                       │
│  - 删除任何不想保留的内容                   │
│  - 导出你的数据                             │
│                                             │
└─────────────────────────────────────────────┘
```

## 七、实现优先级

### P0（MVP - 核心体验）
- [ ] 首次引导（问主人的问题）
- [ ] 主人画像展示页面
- [ ] 基础的对话学习（记录用户反馈）
- [ ] 纠正机制（用户可以纠正理解）

### P1（增强了解）
- [ ] 主动提问机制
- [ ] 定期总结
- [ ] 了解程度可视化
- [ ] 成长时间线

### P2（深度理解）
- [ ] 从行为模式学习

## 八、与旧设计的对比

| 维度 | 旧设计 | 新设计 |
|------|--------|--------|
| 核心 | 用户定义 AI | AI 了解用户 |

| 更新 | 用户主动编辑 | AI 持续学习 |
