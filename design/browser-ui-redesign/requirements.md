# 浏览器 UI 重设计需求

## 背景

当前 Lumos 浏览器工作区的 UI 存在以下问题：
1. **工作区间太小**：右侧固定 360px 侧边栏压缩了浏览器显示区域
2. **布局不合理**：功能模块（Context、Workflows、Downloads）直接堆在右侧，占用大量空间
3. **不符合用户习惯**：与 Chrome 等主流浏览器的全屏体验差距较大

## 设计目标

### 核心原则
- **极简风格**：去除冗余元素，聚焦核心功能
- **极致体验**：流畅、直观、符合用户习惯
- **空间利用**：最大化浏览器工作区，功能模块按需显示

### 参考标准
- Google Chrome：全屏浏览体验，工具栏简洁
- Arc Browser：侧边栏可收起，空间利用高效
- Safari：极简工具栏，沉浸式浏览

## 功能需求

### 必须保留的功能
1. **浏览器核心**
   - 地址栏（URL 输入 + 搜索）
   - 前进/后退/刷新
   - 多 Tab 管理
   - 页面加载状态

2. **AI 功能**
   - AI 活动横幅（当前正在执行的操作）
   - 与 AI 共享页面（pageId 绑定）

3. **上下文采集**
   - 浏览历史
   - 页面事件（导航、加载、错误）
   - 采集设置（暂停/清空/保留期）

4. **Workflow 系统**
   - Workflow 列表
   - 录制/回放
   - 参数化配置

5. **下载管理**
   - 下载列表
   - 下载进度

### 布局优化方向

#### 方案 A：Chrome 风格（推荐）
- **顶部工具栏**：地址栏 + Tab 栏 + 功能按钮
- **全屏浏览区**：最大化显示网页内容
- **侧边抽屉**：功能模块收起在侧边，点击展开（类似 DevTools）
- **底部状态栏**：显示 AI 活动、下载进度等

#### 方案 B：Arc 风格
- **左侧边栏**：可收起，包含 Tab 列表 + 功能入口
- **中间浏览区**：全屏显示网页
- **右侧面板**：按需弹出，显示 Context/Workflows

#### 方案 C：混合风格
- **顶部**：极简工具栏（地址栏 + 核心按钮）
- **浮动面板**：功能模块以浮动窗口形式出现，可拖拽、最小化
- **全屏模式**：支持完全隐藏所有 UI，只显示网页

## 设计团队分工

### 产品经理（Product Manager）
- 分析用户需求和使用场景
- 定义功能优先级
- 制定产品方案

### UI 设计师（UI Designer）
- 设计视觉风格（配色、字体、图标）
- 绘制高保真原型
- 输出设计规范

### 交互设计师（Interaction Designer）
- 设计交互流程
- 定义动画效果
- 优化操作路径

### 用户体验设计师（UX Designer）
- 用户研究和测试
- 信息架构设计
- 可用性评估

### 视觉设计师（Visual Designer）
- 精细化视觉呈现
- 图标和插画设计
- 品牌一致性把控

## 交付物

1. **产品方案文档**：功能定义、优先级、用户故事
2. **交互原型**：Figma/Sketch 原型，包含交互流程
3. **视觉设计稿**：高保真 UI 设计，包含各种状态
4. **设计规范**：组件库、颜色、字体、间距等
5. **实现建议**：技术可行性分析、组件拆分建议

## 时间计划

- **第 1 阶段**（1-2 天）：需求分析 + 竞品研究
- **第 2 阶段**（2-3 天）：方案设计 + 原型制作
- **第 3 阶段**（1-2 天）：视觉设计 + 规范输出
- **第 4 阶段**（1 天）：评审 + 优化

## 成功标准

- ✅ 浏览器工作区占屏幕 80% 以上
- ✅ 功能模块可按需显示/隐藏
- ✅ 操作路径减少 30% 以上
- ✅ 视觉风格统一、现代、简洁
- ✅ 符合主流浏览器使用习惯
