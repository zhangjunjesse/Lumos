# 轻量级工作流引擎对比

## 候选方案

### 1. OpenWorkflow ⭐ 推荐

**GitHub**: `openworkflowdev/openworkflow` (1.2k stars)
**官网**: https://openworkflow.dev
**许可证**: 开源

**核心特性**：
- ✅ **无需额外服务器**：使用 PostgreSQL 或 SQLite 存储状态
- ✅ **持久化和可恢复**：工作流可以在崩溃后自动恢复
- ✅ **TypeScript 原生**：完整的类型安全和自动补全
- ✅ **自动重试**：支持指数退避
- ✅ **并行执行**：支持并行步骤
- ✅ **内置监控面板**：可视化工作流执行状态
- ✅ **步骤记忆化**：避免重复执行

**架构**：
```typescript
import { Workflow, step } from '@openworkflow/core';

const myWorkflow = new Workflow('my-workflow')
  .step('fetch', async () => {
    return await fetchData();
  })
  .step('process', async ({ fetch }) => {
    return await processData(fetch);
  })
  .step('save', async ({ process }) => {
    await saveResult(process);
  });

// 执行
await myWorkflow.run({ input: 'data' });
```

**优点**：
- 社区规模适中，有一定验证
- 支持 SQLite（Lumos 已有）
- 功能完整，开箱即用
- 有监控面板，便于调试

**缺点**：
- 相对较新的项目
- 文档可能不如 Temporal 完善

**适合 Lumos**：⭐⭐⭐⭐⭐

### 2. Flowcraft

**GitHub**: `gorango/flowcraft`
**发布时间**: 2025年10月
**许可证**: MIT

**核心特性**：
- ✅ **零依赖**：完全无依赖，极致轻量
- ✅ **声明式工作流**：DAG 定义
- ✅ **TypeScript 类型安全**：完整类型支持
- ✅ **可扩展**：支持 BullMQ、AWS、GCP、Azure 等适配器
- ✅ **弹性执行**：重试、回退、超时、取消
- ✅ **工具链**：静态分析、循环检测、Mermaid 图表生成
- ✅ **编译器**：可将命令式代码转换为声明式工作流

**架构**：
```typescript
import { createEngine } from 'flowcraft';

const workflow = {
  nodes: {
    fetch: { fn: fetchData },
    process: { fn: processData },
    save: { fn: saveResult }
  },
  edges: [
    { from: 'fetch', to: 'process' },
    { from: 'process', to: 'save' }
  ]
};

const engine = createEngine();
await engine.run(workflow, { input: 'data' });
```

**优点**：
- 零依赖，最轻量
- 声明式 DAG，易于 LLM 生成
- 可视化工具（Mermaid）
- 编译器支持

**缺点**：
- **非常新**（2025年10月），生产验证不足
- 社区规模未知
- 可能存在未发现的 bug

**适合 Lumos**：⭐⭐⭐⭐（如果愿意承担早期采用者风险）

### 3. DBOS Transact

**GitHub**: `dbos-inc/dbos-transact-ts`
**官网**: https://dbos.dev
**最新版本**: 4.0 (2025年12月)

**核心特性**：
- ✅ **持久化执行**：至少执行一次，崩溃后恢复
- ✅ **事务性执行**：恰好执行一次
- ✅ **工作流编排**：支持分支、循环、子任务、重试
- ✅ **持久化队列**：Postgres 支持的队列
- ✅ **幂等性**：请求幂等
- ✅ **调度**：支持 cron 任务

**架构**：
```typescript
import { Workflow, Step } from '@dbos-inc/dbos-sdk';

class MyWorkflow {
  @Workflow()
  static async myWorkflow(ctx: WorkflowContext, input: string) {
    const data = await ctx.invoke(MyWorkflow).fetchStep(input);
    const result = await ctx.invoke(MyWorkflow).processStep(data);
    await ctx.invoke(MyWorkflow).saveStep(result);
  }

  @Step()
  static async fetchStep(ctx: StepContext, input: string) {
    return await fetchData(input);
  }
}
```

**优点**：
- 活跃维护（2025年12月更新）
- 强一致性和事务性
- 企业级特性（DBOS Cloud）

**缺点**：
- **需要 PostgreSQL**（不确定是否支持 SQLite）
- 相对重量级（需要数据库）
- 学习曲线较陡

**适合 Lumos**：⭐⭐⭐（如果能确认支持 SQLite）

## 集成方案（以 OpenWorkflow 为例）

### 1. 安装
```bash
npm install @openworkflow/core @openworkflow/sqlite
```

### 2. 初始化
```typescript
// src/lib/workflow/engine.ts
import { WorkflowEngine } from '@openworkflow/core';
import { SQLiteAdapter } from '@openworkflow/sqlite';
import { getDb } from '@/lib/db';

export function createWorkflowEngine() {
  const db = getDb();
  const adapter = new SQLiteAdapter(db);
  return new WorkflowEngine({ adapter });
}
```

### 3. 定义工作流
```typescript
// src/lib/workflow/workflows/agent-workflow.ts
import { Workflow, step } from '@openworkflow/core';

export const agentWorkflow = new Workflow('agent-workflow')
  .step('analyze', async ({ input }) => {
    // 调用 claude-agent-sdk 分析任务
    return await analyzeTask(input);
  })
  .step('execute', async ({ analyze }) => {
    // 执行具体操作
    return await executeTask(analyze);
  })
  .step('notify', async ({ execute }) => {
    // 发送通知
    await sendNotification(execute);
  });
```

### 4. 执行工作流
```typescript
// src/lib/task-management/api.ts
import { createWorkflowEngine } from '@/lib/workflow/engine';
import { agentWorkflow } from '@/lib/workflow/workflows/agent-workflow';

function submitTask(request: SubmitTaskRequest): SubmitTaskResponse {
  const engine = createWorkflowEngine();

  // 异步执行工作流
  engine.run(agentWorkflow, { input: request.task })
    .then(result => {
      updateTaskStatus({
        taskId: request.taskId,
        status: TaskStatus.COMPLETED,
        result
      });
    })
    .catch(error => {
      updateTaskStatus({
        taskId: request.taskId,
        status: TaskStatus.FAILED,
        errors: [error.message]
      });
    });

  return { accepted: true };
}
```

### 5. LLM 生成工作流

## 总结

**推荐使用 OpenWorkflow**，理由：
- ✅ 轻量级（无需额外服务器）
- ✅ 支持 SQLite（Lumos 已有）
- ✅ 功能完整（持久化、并行、监控）
- ✅ 社区验证（1.2k stars）
- ✅ TypeScript 原生
- ✅ 开发体验好

**风险可控**：
- 先做 POC 验证（1周）
- 如果不满足需求，可以切换到 Flowcraft 或 Temporal

**不推荐自己开发**：
- 工作流引擎复杂度高
- 开发成本大（9-11周）
- 稳定性风险高
