# Temporal 工作流 UI 设计方案

## 1. 设计目标

为 Lumos 用户提供直观、强大的工作流编辑和管理界面，支持：
- 可视化拖拽编辑
- 代码级精确控制
- 实时调试和测试
- 版本管理和协作

**设计原则**：
- 降低学习曲线（对非技术用户友好）
- 保留专业能力（对开发者强大）
- 渐进式复杂度（简单任务简单做，复杂任务能做到）

---

### 3.2 节点设计规范

**基础节点样式**：

```
┌─────────────────────┐
│  🤖 Agent 节点      │  ← 图标 + 类型
├─────────────────────┤
│  角色: 数据分析师   │  ← 关键配置预览
│  任务: 分析用户数据 │
└─────────────────────┘
      │
      ▼ (连接线)
```

**节点状态指示**：
- 🟢 绿色边框：执行成功
- 🔴 红色边框：执行失败
- 🟡 黄色边框：执行中
- ⚪ 灰色边框：未执行

**节点交互**：
- 单击：选中并显示属性面板
- 双击：快速编辑核心配置
- 右键：上下文菜单（复制/删除/禁用）
- 拖拽：移动位置
- 拖拽端点：创建连接

### 3.3 连接线设计

**连接类型**：
- 实线：正常流程
- 虚线：条件分支
- 粗线：数据传递

**连接操作**：
- 悬停显示数据流信息
- 单击选中（可删除）
- 支持贝塞尔曲线自动避让

### 3.4 画布操作

**工具栏**（顶部）：
```
[选择] [拖拽] [连接] | [撤销] [重做] | [放大] [缩小] [适应] | [对齐] [分布]
```

**快捷键**：
- `Space + 拖拽`：平移画布
- `Ctrl + 滚轮`：缩放
- `Ctrl + Z/Y`：撤销/重做
- `Delete`：删除选中节点
- `Ctrl + C/V`：复制粘贴
- `Ctrl + A`：全选

## 4. 代码编辑模式

### 4.1 编辑器布局

```
┌─────────────────────────────────────────────────────────────┐
│  workflow.ts                                    [格式化][验证]│
├─────────────────────────────────────────────────────────────┤
│  1  import { defineWorkflow } from '@lumos/workflow';       │
│  2                                                           │
│  3  export default defineWorkflow({                         │
│  4    name: '数据采集流程',                                 │
│  5    nodes: [                                              │
│  6      {                                                   │
│  7        id: 'agent1',                                     │
│  8        type: 'agent',                                    │
│  9        config: {                                         │
│ 10          role: '数据分析师',                             │
│ 11          task: '分析用户数据'                            │
│ 12        }                                                 │
│ 13      }                                                   │
│ 14    ],                                                    │
│ 15    edges: [                                              │
│ 16      { from: 'start', to: 'agent1' }                    │
│ 17    ]                                                     │
│ 18  });                                                     │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 编辑器特性

**语法支持**：
- TypeScript 语法高亮
- 自动补全（节点类型、配置项）
- 类型检查（实时错误提示）
- 代码片段（快速插入常用模式）

**智能提示**：
```typescript
nodes: [
  {
    type: 'agent',  // ← 输入时显示：agent | llm | http | condition
    config: {
      role: '',     // ← 显示可用角色列表
    }
  }
]
```

**实时验证**：
- 语法错误：红色波浪线
- 类型错误：黄色波浪线
- 循环依赖检测：错误提示

### 4.3 可视化 ↔ 代码同步

**双向同步**：
- 可视化编辑 → 自动生成代码
- 代码编辑 → 自动更新可视化
- 冲突时以当前编辑模式为准

**切换按钮**：
```
[📊 可视化] [</> 代码]
```

## 8. 技术实现方案

### 8.1 前端技术栈

**核心库**：
- React Flow：可视化编辑器基础
- Monaco Editor：代码编辑器
- Zustand：状态管理
- React Query：数据获取

**UI 组件**：
- shadcn/ui：基础组件
- Tailwind CSS：样式
- Framer Motion：动画

### 8.2 数据结构

**工作流定义**：
```typescript
interface WorkflowDefinition {
  id: string;
  name: string;
  description?: string;
  version: string;
  nodes: WorkflowNode[];
  edges: WorkflowEdge[];
  variables?: Record<string, any>;
  metadata?: {
    createdAt: string;
    updatedAt: string;
    author?: string;
    tags?: string[];
  };
}

interface WorkflowNode {
  id: string;
  type: 'agent' | 'llm' | 'condition' | 'loop' | 'http';
  position: { x: number; y: number };
  config: Record<string, any>;
  retry?: {
    maxAttempts: number;
    backoff: number;
  };
}

interface WorkflowEdge {
  id: string;
  source: string;
  target: string;
  condition?: string;
  label?: string;
}
```

### 8.3 API 接口

**工作流管理**：
```typescript
// 创建工作流
POST /api/workflows
Body: { name, description, definition }

// 更新工作流
PUT /api/workflows/:id
Body: { definition }

// 获取工作流列表
GET /api/workflows
Query: { search, tags, page, limit }

// 获取工作流详情
GET /api/workflows/:id

// 删除工作流
DELETE /api/workflows/:id
```

**执行管理**：
```typescript
// 运行工作流
POST /api/workflows/:id/execute
Body: { inputs, debug: boolean }

// 获取执行状态
GET /api/executions/:executionId

// 停止执行
POST /api/executions/:executionId/stop

// 获取执行日志
GET /api/executions/:executionId/logs
```

### 8.4 实时通信

**WebSocket 事件**：

```typescript

        type: 'agent',
        config: { task: `处理 ${item}` }

    }
  }
}
```

## 12. 关键交互流程

### 12.1 创建工作流

```
用户操作                    系统响应
─────────────────────────────────────────
1. 点击"新建工作流"      → 显示模板选择对话框
2. 选择"空白工作流"      → 打开编辑器，显示空画布
3. 从节点库拖拽Agent节点  → 节点出现在画布上
4. 双击节点              → 打开快速配置对话框
5. 输入角色和任务        → 保存配置，更新节点显示
6. 拖拽连接线            → 创建节点间的连接
7. 点击"保存"            → 验证工作流，保存到数据库
```

### 12.2 运行和调试

```
用户操作                    系统响应
─────────────────────────────────────────
1. 点击"运行"            → 显示输入参数对话框
2. 输入测试数据          → 开始执行工作流
3. 观察节点状态变化      → 实时更新节点颜色和进度
4. 点击执行中的节点      → 显示当前输入输出数据
5. 发现问题，点击"暂停"  → 暂停执行，保持当前状态
6. 修改节点配置          → 热更新配置
7. 点击"继续"            → 从暂停点继续执行
8. 执行完成              → 显示完整执行报告
```

### 12.3 版本管理

```
用户操作                    系统响应
─────────────────────────────────────────
1. 修改工作流            → 自动保存草稿
2. 点击"提交版本"        → 显示版本信息对话框
3. 输入版本说明          → 创建新版本（v1.2）
4. 继续修改              → 在 v1.2 基础上修改
5. 点击"版本历史"        → 显示所有历史版本
6. 选择 v1.1 查看        → 只读模式显示旧版本
7. 点击"回滚到此版本"    → 确认后恢复到 v1.1
```
