#### 任务 #16：资料库后端 API 集成
**工期**：5-7 天
**负责人**：后端工程师

**交付物**：
1. 数据库表结构设计
   ```sql
   CREATE TABLE library_items (
     id UUID PRIMARY KEY,
     user_id UUID NOT NULL,
     type VARCHAR(20) NOT NULL, -- 'document', 'conversation', 'audio', 'video', 'web'
     title VARCHAR(255) NOT NULL,
     content TEXT,
     preview TEXT,
     metadata JSONB, -- 存储额外信息（来源、时长、文件大小等）
     tags JSONB, -- 标签数组
     is_favorite BOOLEAN DEFAULT FALSE,
     is_archived BOOLEAN DEFAULT FALSE,
     created_at TIMESTAMP DEFAULT NOW(),
     updated_at TIMESTAMP DEFAULT NOW()
   );

   CREATE INDEX idx_library_items_user_id ON library_items(user_id);
   CREATE INDEX idx_library_items_type ON library_items(type);
   CREATE INDEX idx_library_items_created_at ON library_items(created_at DESC);
   ```

2. API 端点实现
   - `GET /api/library/items` - 获取资料列表（支持分页、筛选、搜索）
   - `GET /api/library/items/[id]` - 获取资料详情
   - `POST /api/library/items` - 创建新资料
   - `PATCH /api/library/items/[id]` - 更新资料
   - `DELETE /api/library/items/[id]` - 删除资料
   - `GET /api/library/tags` - 获取所有标签及使用次数

3. 前端集成
   - 替换 mock 数据为真实 API 调用
   - 实现加载状态和错误处理
   - 实现分页加载（无限滚动）

**验收标准**：
- [ ] 数据库表创建成功
- [ ] 所有 API 端点正常工作
- [ ] 前端页面显示真实数据
- [ ] 分页加载正常
- [ ] 错误处理完善

#### 任务 #22：RAG 知识库功能
**工期**：7-10 天
**负责人**：AI 工程师 + 后端工程师

**目标**：实现向量检索和知识库功能

**技术方案**：
1. 向量化
   - 使用 OpenAI Embedding API
   - 分块策略（每块 500-1000 tokens）
   - 异步处理（后台任务）
2. 向量数据库
   - 选择：Pinecone / Qdrant / Weaviate
   - 索引创建和管理
   - 相似度搜索
3. 语义搜索
   - 替代传统关键词搜索
   - 支持自然语言查询
   - 结果排序和过滤
4. 智能推荐
   - 基于当前资料推荐相关内容
   - 基于用户行为推荐
   - 显示相似度分数
5. RAG 对话
   - AI 基于资料库回答问题
   - 引用来源
   - 上下文窗口管理

**验收标准**：
- [ ] 向量化成功率 > 95%
- [ ] 语义搜索准确率 > 80%
