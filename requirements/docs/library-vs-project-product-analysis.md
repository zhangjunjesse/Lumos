# 资料库（Library）vs 项目（Project）产品分析

## 执行摘要

本文档从产品经理角度分析 Lumos 中"资料库"和"项目"两个核心概念的产品定位、用户故事和使用场景。

**核心结论**：
- **资料库**：个人知识资产的长期存储中心，关键词"我拥有什么"
- **项目**：目标导向的临时工作空间，关键词"我在做什么"
- **关系**：通过引用和归档形成闭环

---

## 1. 核心定位差异

### 资料库（Library）

**定义**：个人知识资产的长期存储中心

**本质**：内容仓库（Content Repository）

**核心价值**：
- 知识沉淀：长期保存有价值的内容
- 快速检索：随时查找需要的信息
- 跨项目复用：在不同项目中重用知识

**关键词**："我拥有什么"（What I have）

### 项目（Project）

**定义**：目标导向的临时工作空间

**本质**：任务容器（Task Container）

**核心价值**：
- 上下文聚焦：专注于当前任务
- 协作协同：团队成员共同工作
- 进度追踪：监控任务完成情况

**关键词**："我在做什么"（What I'm doing）

## 2. 用户故事对比

### 资料库典型场景（8个）

1. **开发者保存代码片段和技术文档，跨项目复用**
   - 作为开发者，我想保存常用的代码片段
   - 以便在不同项目中快速复用
   - 避免重复编写相同的代码

2. **产品经理收集竞品分析和行业研究，随时查阅**
   - 作为产品经理，我想收集竞品分析报告
   - 以便在制定产品策略时参考
   - 保持对行业动态的了解

3. **设计师整理设计规范和灵感素材，建立资产库**
   - 作为设计师，我想整理设计规范文档
   - 以便团队成员遵循统一的设计标准
   - 建立可复用的设计资产库

4. **学生归档课程笔记和论文资料，系统复习**
   - 作为学生，我想归档所有课程笔记
   - 以便考试前系统复习
   - 建立个人知识体系

5. **内容创作者保存写作素材和参考文章**
   - 作为内容创作者，我想保存写作灵感和素材
   - 以便创作时快速找到参考
   - 提高创作效率

6. **研究员管理文献和实验数据，长期追踪**
   - 作为研究员，我想管理所有相关文献
   - 以便追踪研究进展
   - 建立系统的知识库

7. **团队负责人维护知识库，帮助新人快速上手**
   - 作为团队负责人，我想维护团队知识库
   - 以便新人快速了解项目背景
   - 降低团队知识传承成本

8. **自由职业者整理客户资料和项目模板**
   - 作为自由职业者，我想整理客户资料
   - 以便快速启动新项目
   - 复用成功的项目模板

**特征**：动词是"保存、收集、整理、归档"，强调长期积累

### 项目典型场景（8个）

1. **开发者为"重构支付模块"创建项目，集中管理代码和文档**
   - 作为开发者，我想为重构任务创建独立项目
   - 以便集中管理相关代码和文档
   - 避免与其他工作混淆

2. **产品经理为"Q2 产品规划"创建项目，跟踪需求和评审**
   - 作为产品经理，我想为季度规划创建项目
   - 以便跟踪所有需求和评审进度
   - 确保按时完成规划

3. **设计师为"品牌升级设计"创建项目，组织设计稿和迭代**
   - 作为设计师，我想为品牌升级创建项目
   - 以便组织所有设计稿和迭代记录
   - 方便团队协作和反馈

4. **学生为"毕业论文"创建项目，管理文献和草稿**
   - 作为学生，我想为毕业论文创建项目
   - 以便管理所有相关文献和草稿
   - 追踪论文写作进度

5. **内容创作者为"新书写作"创建项目，规划章节和素材**
   - 作为内容创作者，我想为新书创建项目
   - 以便规划章节结构和素材
   - 追踪写作进度

6. **研究员为"实验 A"创建项目，记录步骤和数据**
   - 作为研究员，我想为每个实验创建项目
   - 以便记录实验步骤和数据
   - 隔离不同实验的内容

7. **团队负责人为"客户 X 项目"创建项目，协调交付**
   - 作为团队负责人，我想为客户项目创建独立空间
   - 以便协调团队成员工作
   - 追踪项目交付进度

8. **自由职业者为每个订单创建项目，隔离工作内容**
   - 作为自由职业者，我想为每个订单创建项目
   - 以便隔离不同客户的工作内容
   - 避免信息混淆

**特征**：动词是"创建、跟踪、组织、完成"，强调目标导向

## 3. 内容类型与流转

### 资料库内容

**内容类型**：
- 文档（Word、PDF、Markdown）
- 网页（收藏的网页、文章）
- 笔记（个人笔记、摘录）
- 代码片段（常用代码、模板）
- 媒体（图片、音频、视频）
- 数据（表格、数据集）

**内容特点**：
- 内容稳定，不经常变化
- 按主题或类型组织
- 支持全文搜索和标签分类
- 强调长期保存和复用

### 项目内容

**内容类型**：
- 任务列表（待办事项、里程碑）
- 对话记录（与 AI 的对话历史）
- 工作文件（正在编辑的文件）
- 关联资源（引用的资料库内容）
- 时间线（项目进度、变更历史）

**内容特点**：
- 内容动态，频繁变化
- 按工作流程组织
- 强调上下文连续性
- 项目完成后可归档

### 内容流转闭环

```
创建内容 → 保存到资料库
    ↓
启动项目 → 引用资料库内容
    ↓
完成项目 → 归档到资料库
    ↓
删除项目 → 资料库内容保留
```

**关键点**：
1. 资料库是持久化存储，项目是临时工作空间
2. 项目可以引用资料库内容（单向引用）
3. 项目完成后，有价值的内容归档到资料库
4. 删除项目不影响资料库内容

## 4. 使用场景协同

### 场景示例 1：开发新功能

**步骤**：
1. 在资料库搜索"API 设计最佳实践"
2. 创建项目"用户认证模块"
3. 在项目中引用资料库文档作为参考
4. 开发过程中与 AI 对话，记录在项目中
5. 完成后将 API 设计文档归档到资料库
6. 归档项目，保留对话记录供未来参考

### 场景示例 2：撰写研究报告

**步骤**：
1. 在资料库收集相关文献和笔记
2. 创建项目"Q1 市场研究报告"
3. 在项目中引用资料库的文献
4. 使用 AI 辅助分析和写作
5. 完成后将报告归档到资料库
6. 归档项目，删除临时草稿

### 场景示例 3：学习新技术

**步骤**：
1. 在资料库保存学习资源（教程、文档）
2. 创建项目"学习 React Hooks"
3. 在项目中实践代码，记录学习笔记
4. 完成后将笔记和代码片段归档到资料库
5. 归档项目，资料库成为个人知识库

## 5. 信息架构建议

### 当前问题

**侧边栏项目过多**：
- AI 助手、资料库、最近、浏览器、收藏、回收站、知识库、扩展
- 用户可能混淆"资料库"和"知识库"
- 项目列表与导航项混在一起

### 优化方案

**简化侧边栏**：
```
侧边栏（简化）
├── 🏠 首页
├── 💬 AI 助手
├── 📚 资料库
│   ├── 文档
│   ├── 网页
│   └── 知识库（RAG）
├── 🔌 扩展
└── 🗑️ 回收站

底部
└── 📁 项目
    ├── 进行中（3）
    └── 已归档（12）
```

**关系定义**：
- 资料库和项目是平行的顶层概念
- 项目可以引用资料库内容（单向引用）
- 资料库不感知项目存在（解耦设计）

### 数据模型建议

**资料库表**（library_items）：
```sql
CREATE TABLE library_items (
  id UUID PRIMARY KEY,
  title TEXT NOT NULL,
  content TEXT,
  type TEXT NOT NULL, -- document, webpage, note, code, media
  tags TEXT[],
  source_url TEXT,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

**项目表**（projects）：
```sql
CREATE TABLE projects (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  path TEXT NOT NULL,
  status TEXT NOT NULL, -- active, archived
  created_at TIMESTAMP,
  archived_at TIMESTAMP
);
```

**项目引用表**（project_references）：
```sql
CREATE TABLE project_references (
  id UUID PRIMARY KEY,
  project_id UUID REFERENCES projects(id),
  library_item_id UUID REFERENCES library_items(id),
  created_at TIMESTAMP
);
```

## 6. 产品建议

### 1. 强化差异化体验

**资料库**：
- UI 风格：图书馆风格，强调"永久保存"和"快速检索"
- 交互重点：浏览、搜索、筛选、AI 创作
- 视觉元素：卡片网格、标签云、搜索框

**项目**：
- UI 风格：工作台风格，强调"临时工作空间"和"上下文连续"
- 交互重点：切换、管理、文件浏览
- 视觉元素：列表、激活状态、文件树

### 2. 打通内容流转

**资料库 → 项目**：
- 资料库卡片右键菜单："添加到项目"
- 选择目标项目，创建引用关系
- 项目中显示引用的资料库内容

**项目 → 资料库**：
- 项目完成后弹窗："是否归档有价值的内容？"
- 选择要归档的文件和对话记录
- 自动保存到资料库，添加标签

### 3. 智能推荐

**基于资料库的项目推荐**：
- 创建项目时，AI 推荐相关的资料库内容
- "你可能需要这些资料：[文档 A]、[笔记 B]"

**基于项目的资料库整理**：
- 项目完成后，AI 自动提取有价值的内容
- "以下内容建议归档到资料库：[对话记录]、[代码片段]"

## 7. 核心理念总结

### 产品定位对比

| 维度 | 资料库 | 项目 |
|------|--------|------|
| **定位** | 知识银行 | 工作现场 |
| **时间** | 长期保存 | 临时使用 |
| **组织** | 按内容类型 | 按工作流程 |
| **复用** | 高复用 | 低复用 |
| **UI 风格** | 图书馆 | 工作台 |
| **核心价值** | 知识沉淀 | 任务执行 |

### 用户价值主张

**资料库**：
- "我的知识永远不会丢失"
- "我可以快速找到需要的信息"
- "我的知识可以在不同项目中复用"

**项目**：
- "我可以专注于当前任务"
- "我的工作内容不会与其他项目混淆"
- "我可以追踪项目进度和完成情况"

### 产品哲学

**资料库是"知识银行"**：
- 长期保存有价值的知识资产
- 支持快速检索和跨项目复用
- 形成个人或团队的知识体系

**项目是"工作现场"**：
- 提供临时的工作空间
- 隔离不同任务的上下文
- 完成后归档，保持工作区整洁

**两者通过引用和归档形成闭环**：
- 资料库为项目提供知识支持
- 项目为资料库贡献新知识
- 形成知识积累和任务执行的良性循环

## 8. 实施路线图

### Phase 1：基础功能（3 个月）

**资料库**：
- 基础 UI（卡片网格、搜索、筛选）
- 内容导入（文档、网页、笔记）
- 标签管理

**项目**：
- 项目列表（侧边栏）
- 项目切换（激活状态）
- 项目管理（添加、重命名、删除）

**关联**：
- 项目筛选（资料库页面筛选特定项目）

### Phase 2：智能功能（6 个月）

**资料库**：
- AI 自动标签
- 知识图谱
- 智能推荐

**项目**：
- 项目模板
- 文件树视图
- 协作功能

**关联**：
- 内容引用（项目引用资料库内容）
- 内容归档（项目完成后归档到资料库）

### Phase 3：高级功能（12 个月）

**资料库**：
- 多种视图（列表、时间线、图谱）
- 高级搜索（语义搜索、向量检索）
- 知识演化追踪

**项目**：
- 多人协作
- 版本控制集成
- 项目分析和报告

**关联**：
- 智能推荐（基于资料库推荐项目内容）
- 自动归档（AI 自动识别有价值的内容）

## 9. 成功指标

### 资料库指标

**使用指标**：
- 资料库内容数量
- 搜索使用频率
- 标签使用情况
- AI 创作使用频率

**价值指标**：
- 内容复用率（同一内容被多个项目引用）
- 搜索成功率（用户找到需要的内容）
- 知识沉淀率（新内容添加到资料库的比例）

### 项目指标

**使用指标**：
- 项目数量（进行中 + 已归档）
- 项目切换频率
- 项目完成率

**价值指标**：
- 项目隔离效果（不同项目的内容混淆率）
- 项目完成时间（从创建到归档的平均时间）
- 内容归档率（项目完成后归档到资料库的比例）

### 关联指标

**流转指标**：
- 资料库 → 项目引用率
- 项目 → 资料库归档率
- 内容复用率（资料库内容被多个项目引用）

---

## 10. 结论

**资料库和项目是两个互补而非重复的功能**：
- 资料库是"知识银行"，强调长期保存和跨项目复用
- 项目是"工作现场"，强调临时工作空间和上下文隔离
- 两者通过引用和归档形成闭环，共同支撑 Lumos 的产品愿景

**核心价值主张**：
- Lumos 不是又一个聊天机器人、笔记软件或 IDE
- 而是 AI 原生的知识工作平台
- 资料库 = AI 的长期记忆，项目 = AI 的工作上下文
- 两者结合 = "有记忆、能执行任务的 AI 助手"

**实施建议**：
- 保持资料库和项目的清晰区分
- 强化差异化体验（UI 风格、交互模式）
- 打通内容流转（引用、归档）
- 逐步实现智能功能（推荐、自动归档）
