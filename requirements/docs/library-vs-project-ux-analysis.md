# 资料库（Library）vs 项目（Project）交互设计分析

## 执行摘要

本文档从 UX 设计师角度分析 Lumos 中"资料库"和"项目"两个核心概念的交互设计差异与联系。通过对比业界标杆产品（Notion、Obsidian、VS Code），提出清晰的设计原则和实现建议。

**核心结论**：
- **资料库**：内容中心，强调浏览、搜索、AI 创作
- **项目**：工作上下文，强调切换、隔离、文件管理
- **关系**：项目是资料库的"过滤视图"，资料库是项目的"聚合视图"

---

## 1. 概念定位

### 1.1 资料库（Library）

**定义**：用户所有内容的统一视图，跨项目的知识中心

**核心价值**：
- 打破项目边界，发现内容关联
- AI 辅助创作的起点
- 知识沉淀与复用

**用户心智模型**：
- "我的所有资料在哪里？"
- "我之前写过类似的东西吗？"
- "让 AI 帮我基于这些资料创作"

### 1.2 项目（Project）

**定义**：工作目录的抽象，提供隔离的工作上下文

**核心价值**：
- 隔离不同工作场景（工作/个人/学习）
- 提供文件系统级别的上下文
- 快速切换工作环境

**用户心智模型**：
- "我现在在哪个项目里工作？"
- "这个项目有哪些文件？"
- "切换到另一个项目"

## 2. 呈现方式（Presentation）

### 2.1 资料库页面

#### 推荐布局：卡片网格（Card Grid）

**理由**：
1. **信息密度适中**：每张卡片展示标题、预览、标签、元信息
2. **视觉扫描效率高**：用户可快速浏览大量内容
3. **支持多种内容类型**：文档、对话、音视频、网页统一呈现
4. **符合现代审美**：Pinterest、Notion Gallery 已验证的模式

**卡片信息层级**（从上到下）：
```
┌─────────────────────────────┐
│ [Logo] 标题（2行截断）       │
│        预览文本（2行截断）   │
│ 📎 路径/来源                 │
│ #标签1 #标签2 #标签3 +2     │
│ 文档类型          2小时前    │
│ [悬停：查看/标签/更多]       │
└─────────────────────────────┘
```

**网格规格**：
- 3列布局（1920px 屏幕）
- 2列布局（1280px 屏幕）
- 1列布局（<1024px 屏幕）
- 卡片间距：16px
- 卡片圆角：12px
- 悬停效果：边框高亮 + 阴影提升

#### 替代方案：时间线（Timeline）

**适用场景**：
- 用户需要按时间回溯内容
- 强调"最近使用"的场景

**不推荐理由**：
- 信息密度低，需要大量滚动
- 不适合跨时间段的内容发现
- 难以支持多维度筛选

### 2.2 项目列表

#### 推荐布局：侧边栏列表（Sidebar List）

**理由**：
1. **持久可见**：始终在侧边栏，无需导航
2. **快速切换**：一键激活项目
3. **状态清晰**：高亮显示当前激活项目
4. **节省空间**：折叠时仅显示图标

**列表项信息**（展开状态）：
```
┌─────────────────────────┐
│ 📁 项目名称              │
│    [重命名] [删除]       │  ← 悬停显示
└─────────────────────────┘
```

**交互状态**：
- **激活状态**：背景高亮 + 字体加粗
- **悬停状态**：显示操作按钮（重命名/删除）
- **折叠状态**：仅显示文件夹图标

## 3. 交互模式（Interaction Patterns）

### 3.1 资料库核心交互

#### 3.1.1 浏览（Browse）

**主要交互**：
- 滚动浏览卡片网格
- 点击卡片查看详情
- 悬停显示快捷操作

**设计要点**：
- 无限滚动 vs 分页：推荐**虚拟滚动**（性能 + 体验）
- 加载状态：骨架屏（Skeleton）而非 Spinner
- 空状态：引导用户添加内容或切换筛选

#### 3.1.2 搜索（Search）

**搜索范围**：
- 标题
- 预览文本
- 标签
- 路径/来源

**搜索体验**：
- 实时搜索（debounce 300ms）
- 高亮匹配关键词
- 显示结果计数："找到 12 条结果"
- 支持快捷键：`Cmd+F` 聚焦搜索框

#### 3.1.3 筛选（Filter）

**筛选维度**：
1. **类型筛选**：文档/对话/音频/视频/网页
2. **标签筛选**：多选，OR 逻辑（包含任意标签即可）
3. **时间筛选**：今天/本周/本月/自定义
4. **项目筛选**：显示特定项目的内容

**筛选 UI**：
- 类型筛选：顶部 Tab 栏（带 Emoji 图标）
- 标签筛选：下拉面板 + 已选标签显示
- 时间筛选：下拉菜单
- 项目筛选：下拉菜单（与侧边栏项目列表联动）

#### 3.1.4 AI 创作（AI Compose）

**入口位置**：底部固定输入框

**交互流程**：
1. 用户输入问题/指令
2. 对话记录展开（可折叠）
3. AI 流式回复
4. 支持追问和上下文对话

**设计要点**：
- 输入框始终可见（固定在底部）
- 对话记录可折叠（节省空间）
- 支持引用资料库内容（@提及）
- 支持保存对话到资料库

### 3.2 项目核心交互

#### 3.2.1 切换（Switch）

**主要交互**：
- 点击项目列表项激活
- 激活后自动跳转到该项目的最近会话

**设计要点**：
- 切换动画：淡入淡出（200ms）
- 状态持久化：记住最后激活的项目
- 快捷键：`Cmd+1~9` 切换前 9 个项目

#### 3.2.2 管理（Manage）

**主要交互**：
- 添加项目：点击 `+` 按钮，选择文件夹
- 重命名项目：悬停显示编辑按钮
- 删除项目：悬停显示删除按钮，需二次确认

**设计要点**：
- 添加项目：支持拖拽文件夹到侧边栏
- 重命名：内联编辑（inline edit）
- 删除：确认对话框说明"仅移除项目，不删除文件"

#### 3.2.3 文件浏览（File Browse）

**当前缺失**：项目内的文件浏览功能

**建议实现**：
- 点击项目名称展开文件树
- 支持搜索项目内文件
- 支持快速打开文件

## 4. 导航逻辑

### 4.1 用户如何在资料库和项目之间切换？

#### 方案 A：侧边栏导航（推荐）

**实现**：
- 侧边栏顶部：资料库导航项（点击进入资料库页面）
- 侧边栏中部：项目列表（点击激活项目，跳转到对话页面）

**优点**：
- 清晰的层级关系
- 符合用户心智模型（资料库 = 全局视图，项目 = 局部视图）
- 无需额外导航步骤

**缺点**：
- 资料库和项目的关系不够直观

#### 方案 B：面包屑导航

**实现**：
- 页面顶部显示：`资料库 > 项目A`
- 点击"资料库"返回全局视图
- 点击"项目A"进入项目视图

**优点**：
- 明确显示当前位置
- 支持快速返回上级

**缺点**：
- 占用顶部空间
- 增加导航复杂度

**推荐**：采用方案 A，辅以方案 B（在资料库页面显示当前筛选的项目）

### 4.2 资料库中的内容能否关联到项目？

**答案**：是，通过"项目筛选"功能

**实现**：
- 资料库页面顶部添加"项目筛选"下拉菜单
- 选择项目后，仅显示该项目的内容
- 面包屑显示：`资料库 > 项目A`

**设计要点**：
- 默认显示所有项目的内容
- 筛选后显示项目名称和内容计数
- 支持快速切换项目

### 4.3 项目中的内容能否添加到资料库？

**答案**：自动添加，无需手动操作

**实现逻辑**：
- 项目中的所有文件自动索引到资料库
- 对话记录自动保存到资料库
- 用户无需感知"添加"操作

**设计要点**：
- 资料库是"聚合视图"，自动包含所有项目内容
- 用户可通过标签、类型、项目筛选来组织内容

## 5. 状态管理

### 5.1 资料库的"当前选中"

**问题**：资料库有"当前选中"的概念吗？

**答案**：有，但不是"激活状态"，而是"查看状态"

**状态类型**：
1. **查看状态**：点击卡片后，进入详情页面
2. **筛选状态**：当前应用的筛选条件（类型/标签/时间/项目）
3. **搜索状态**：当前搜索关键词

**持久化**：
- 查看状态：URL 参数（`/library?item=xxx`）
- 筛选状态：URL 参数 + LocalStorage
- 搜索状态：仅 URL 参数（不持久化）

### 5.2 项目的"激活状态"

**问题**：项目有"激活状态"（active）

**答案**：是，且是全局唯一的

**状态类型**：
1. **激活状态**：当前工作的项目（全局唯一）
2. **悬停状态**：鼠标悬停时显示操作按钮
3. **编辑状态**：重命名时的内联编辑

**持久化**：
- 激活状态：数据库（`workspaces.is_active`）+ LocalStorage
- 悬停状态：不持久化
- 编辑状态：不持久化

### 5.3 两者的状态管理差异

| 维度 | 资料库 | 项目 |
|------|--------|------|
| **全局唯一性** | 否（可同时查看多个内容） | 是（只能激活一个项目） |
| **持久化** | URL + LocalStorage | 数据库 + LocalStorage |
| **影响范围** | 仅资料库页面 | 全局（影响对话上下文） |
| **状态类型** | 查看/筛选/搜索 | 激活/悬停/编辑 |

## 6. 相似性与差异性

### 6.1 UI 一致性

**应该保持一致的元素**：
1. **色彩系统**：主色、辅助色、状态色
2. **字体系统**：字号、行高、字重
3. **圆角规范**：按钮、卡片、输入框
4. **间距系统**：4px 基准，8/12/16/24/32px
5. **图标风格**：统一使用 Hugeicons

**可以不同的元素**：
1. **布局方式**：资料库用网格，项目用列表
2. **信息密度**：资料库信息丰富，项目简洁
3. **交互模式**：资料库强调浏览，项目强调切换

### 6.2 交互模式一致性

**应该相同的交互**：
1. **悬停反馈**：统一的悬停效果（背景色变化）
2. **点击反馈**：统一的点击动画（缩放/涟漪）
3. **加载状态**：统一的骨架屏/Spinner
4. **错误提示**：统一的 Toast/Dialog

**应该不同的交互**：
1. **主要操作**：资料库是"查看"，项目是"激活"
2. **次要操作**：资料库是"标签/搜索"，项目是"重命名/删除"
3. **导航方式**：资料库是"页面跳转"，项目是"状态切换"

## 7. 参考案例分析

### 7.1 Notion

**资料库类比**：Notion 的"All Pages"视图
- 网格/列表/时间线多种视图
- 强大的筛选和排序
- 支持数据库视图（Table/Board/Calendar）

**项目类比**：Notion 的"Workspace"
- 侧边栏显示 Workspace 列表
- 点击切换 Workspace
- 每个 Workspace 有独立的页面树

**启示**：
- 资料库应支持多种视图（网格/列表/时间线）
- 项目应提供清晰的切换入口
- 两者通过筛选器关联

### 7.2 Obsidian

**资料库类比**：Obsidian 的"Graph View"
- 可视化笔记关联
- 支持搜索和筛选
- 强调知识发现

**项目类比**：Obsidian 的"Vault"
- 每个 Vault 是独立的笔记库
- 切换 Vault 需要重启应用
- Vault 内有文件树

**启示**：
- 资料库应强调内容关联（标签/链接）
- 项目切换应轻量化（无需重启）
- 项目内应提供文件浏览

### 7.3 VS Code

**资料库类比**：VS Code 的"Search"功能
- 全局搜索所有文件
- 支持正则和筛选
- 显示搜索结果列表

**项目类比**：VS Code 的"Workspace"
- 侧边栏显示文件树
- 支持多根工作区
- 切换 Workspace 需要重新打开

**启示**：
- 资料库应提供强大的搜索功能
- 项目应提供文件树视图
- 项目切换应保留上下文

## 8. 设计建议

### 8.1 资料库页面设计

#### 布局结构
```
┌─────────────────────────────────────────────────────────┐
│ [筛选栏] 全部 📄文档 💬对话 🎵音频 🎬视频 🌐网页        │
│          [搜索框]                                        │
├─────────────────────────────────────────────────────────┤
│ [标签筛选] #产品 #设计 #重要 [+添加标签]                │
│ 找到 12 条结果                                           │
├─────────────────────────────────────────────────────────┤
│ ┌─────┐ ┌─────┐ ┌─────┐                                 │
│ │卡片1│ │卡片2│ │卡片3│                                 │
│ └─────┘ └─────┘ └─────┘                                 │
│ ┌─────┐ ┌─────┐ ┌─────┐                                 │
│ │卡片4│ │卡片5│ │卡片6│                                 │
│ └─────┘ └─────┘ └─────┘                                 │
├─────────────────────────────────────────────────────────┤
│ [AI 输入框] 问我任何问题...                              │
│ [对话记录] (可折叠)                                      │
└─────────────────────────────────────────────────────────┘
```

#### 交互细节
1. **筛选栏**：
   - 类型筛选：Tab 切换，带计数
   - 搜索框：右上角，支持快捷键
   - 项目筛选：下拉菜单（可选）

2. **标签筛选**：
   - 已选标签：显示在筛选栏下方，可点击移除
   - 添加标签：点击展开下拉面板，显示所有标签及使用次数
   - 多选逻辑：OR（包含任意标签即可）

3. **卡片网格**：
   - 3列布局（响应式）
   - 虚拟滚动（性能优化）
   - 悬停显示操作按钮

4. **AI 输入框**：
   - 固定在底部
   - 对话记录可折叠
   - 支持流式回复

### 8.2 项目列表设计

#### 布局结构
```
┌─────────────────────┐
│ WORKSPACES      [+] │
├─────────────────────┤
│ 📁 项目A (激活)      │
│ 📁 项目B             │
│ 📁 项目C             │
└─────────────────────┘
```

#### 交互细节
1. **添加项目**：
   - 点击 `+` 按钮
   - 打开文件夹选择器
   - 自动提取文件夹名称作为项目名

2. **激活项目**：
   - 点击项目名称
   - 背景高亮 + 字体加粗
   - 自动跳转到该项目的最近会话

3. **管理项目**：
   - 悬停显示操作按钮（重命名/删除）
   - 重命名：内联编辑
   - 删除：二次确认

### 8.3 关联设计

#### 资料库 → 项目
- 资料库页面添加"项目筛选"下拉菜单
- 选择项目后，仅显示该项目的内容
- 面包屑显示：`资料库 > 项目A`

#### 项目 → 资料库
- 项目列表项右键菜单：`在资料库中查看`
- 点击后跳转到资料库页面，自动应用项目筛选

## 9. 实现优先级

### P0（必须实现）
1. 资料库页面：卡片网格布局
2. 类型筛选：文档/对话/音频/视频/网页
3. 搜索功能：实时搜索 + 结果计数
4. 项目列表：侧边栏显示 + 激活状态
5. 项目切换：点击激活 + 跳转会话

### P1（重要但不紧急）
1. 标签筛选：多选 + 下拉面板
2. AI 输入框：固定底部 + 对话记录
3. 项目筛选：资料库页面筛选项目
4. 悬停操作：卡片悬停显示操作按钮
5. 虚拟滚动：性能优化

### P2（可以延后）
1. 多种视图：列表/时间线视图
2. 时间筛选：今天/本周/本月
3. 文件树：项目内文件浏览
4. 拖拽排序：项目列表拖拽排序
5. 快捷键：Cmd+1~9 切换项目

## 10. 总结

### 核心设计原则

1. **资料库是内容中心，项目是工作上下文**
   - 资料库强调浏览、搜索、AI 创作
   - 项目强调切换、隔离、文件管理

2. **资料库和项目通过筛选关联**
   - 资料库可筛选特定项目的内容
   - 项目可跳转到资料库查看

3. **UI 保持一致，交互有所区别**
   - 色彩、字体、圆角、间距统一
   - 布局、信息密度、交互模式不同

4. **状态管理清晰分离**
   - 资料库：查看/筛选/搜索状态
   - 项目：激活/悬停/编辑状态

### 关键交互流程

1. **用户浏览资料库**：
   - 进入资料库页面 → 浏览卡片网格 → 点击卡片查看详情

2. **用户搜索内容**：
   - 输入搜索关键词 → 实时显示结果 → 点击卡片查看详情

3. **用户筛选内容**：
   - 选择类型/标签/项目 → 显示筛选结果 → 点击卡片查看详情

4. **用户切换项目**：
   - 点击项目列表项 → 激活项目 → 跳转到该项目的最近会话

5. **用户 AI 创作**：
   - 输入问题/指令 → AI 流式回复 → 保存对话到资料库
