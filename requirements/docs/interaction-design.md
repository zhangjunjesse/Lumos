## 一、执行摘要

本文档定义飞书同步功能的完整交互流程，包括用户操作、系统反馈、界面元素和错误处理。

### 1.1 设计目标

1. **简单易用**：用户在 3 步内完成绑定（点击按钮 → 复制链接 → 完成）
2. **状态清晰**：用户随时知道同步状态（已绑定/未绑定/已暂停）
3. **错误友好**：错误提示清晰、可操作
4. **性能感知**：长时间操作有进度提示

### 1.2 核心流程

1. **创建绑定流程**：用户点击按钮 → 显示分享链接 → 复制到飞书 → 绑定成功
2. **查看状态流程**：查看状态徽章 → 点击查看详情 → 查看同步统计
3. **暂停/恢复流程**：点击开关 → 确认操作 → 显示结果
4. **解绑流程**：点击解绑按钮 → 确认对话框 → 解绑成功
5. **错误处理流程**：检测错误 → 显示错误提示 → 提供操作建议

### 2.5 错误处理流程

```mermaid
flowchart TD
    Start[操作触发] --> CallAPI[调用 API]
    CallAPI --> CheckError{检查错误类型}

    CheckError -->|网络错误| NetworkError[显示网络错误提示]

    ShowRetry --> UserRetry{用户操作}

    ShowErrorCode --> UserAction{用户操作}

    OpenHelp --> End
    OpenSettings --> End
    Success --> End
```

## 三、交互细节说明

### 3.1 创建绑定流程

#### 步骤 1：点击"同步到飞书"按钮

**用户操作**：
- 在会话页面右上角找到"同步到飞书"按钮
- 点击按钮

**系统反馈**：
- 按钮变为加载状态（Spinner + "创建中..."）
- 按钮禁用，防止重复点击
- 调用 `POST /api/bridge/bindings` API

**界面元素**：
- 按钮位置：会话页面右上角（标题栏右侧）
- 按钮样式：主要按钮（Primary Button）
- 按钮文字：默认"同步到飞书"，加载时"创建中..."
- 按钮图标：飞书 Logo（可选）

#### 步骤 2：显示分享链接对话框

**用户操作**：
- 查看分享链接
- 点击"复制链接"按钮

**系统反馈**：
- 显示模态对话框（Modal Dialog）
- 对话框标题："同步到飞书"
- 对话框内容：
  - 说明文字："复制以下链接，粘贴到飞书即可完成绑定"
  - 分享链接（只读输入框）
  - "复制链接"按钮
  - "关闭"按钮

**界面元素**：
- 对话框宽度：480px
- 对话框高度：自适应
- 链接输入框：只读，单行，自动选中
- 复制按钮：次要按钮（Secondary Button）
- 关闭按钮：文本按钮（Text Button）

#### 步骤 3：复制链接

**用户操作**：
- 点击"复制链接"按钮

**系统反馈**：
- 链接复制到剪贴板
- 按钮文字变为"已复制"
- 按钮图标变为勾选图标
- 2 秒后恢复为"复制链接"

**界面元素**：
- 按钮状态：默认 → 加载 → 成功 → 默认
- 成功状态持续时间：2 秒
- 成功状态颜色：绿色

#### 步骤 4：绑定完成

**用户操作**：
- 粘贴链接到飞书
- 加入飞书群组

**系统反馈**：
- 会话页面显示"已绑定"徽章
- 移除"同步到飞书"按钮
- 显示成功提示："已成功绑定到飞书"

**界面元素**：
- 徽章位置：会话页面右上角（标题栏右侧）
- 徽章样式：绿色徽章 + 飞书 Logo
- 徽章文字："已绑定"
- 徽章可点击：是

- 无确认对话框

### 3.5 错误处理流程

#### 错误类型 1：网络错误

**触发条件**：
- API 请求超时
- 网络连接失败

**系统反馈**：
- 显示错误提示（Toast）
- 提示内容："网络连接失败，请检查网络后重试"
- 提示类型：错误（红色）
- 提示持续时间：5 秒

**操作建议**：
- 显示"重试"按钮
- 点击后重新调用 API

**界面元素**：
- Toast 位置：屏幕右上角
- Toast 样式：红色背景，白色文字
- 重试按钮：次要按钮（Secondary Button）

#### 错误类型 2：飞书 API 错误

**触发条件**：
- 飞书 API 返回错误码（99991400、230002 等）

**系统反馈**：
- 显示错误提示（Toast）
- 提示内容："飞书服务异常（错误码：99991400）"
- 提示类型：错误（红色）
- 提示持续时间：5 秒

**操作建议**：
- 显示"查看帮助"按钮
- 点击后打开帮助文档（错误码说明）

**界面元素**：
- Toast 位置：屏幕右上角
- Toast 样式：红色背景，白色文字
- 查看帮助按钮：文本按钮（Text Button）

#### 错误类型 3：权限不足

**触发条件**：
- 飞书 API 返回权限错误（230001）

**系统反馈**：
- 显示错误提示（Toast）
- 提示内容："机器人无权限发送消息，请检查权限配置"
- 提示类型：错误（红色）
- 提示持续时间：5 秒

**操作建议**：
- 显示"打开设置"按钮
- 点击后打开飞书设置页面

## 六、边界情况处理

### 6.1 网络错误

**场景**：
- API 请求超时（> 30 秒）
- 网络连接失败
- DNS 解析失败

**处理方式**：
1. 显示错误提示："网络连接失败，请检查网络后重试"
2. 提供"重试"按钮
3. 点击重试后重新调用 API
4. 最多重试 3 次
5. 3 次失败后显示："网络异常，请稍后再试"

**用户体验**：
- 错误提示清晰
- 提供明确的操作建议
- 避免无限重试

### 6.2 飞书 API 错误

**场景**：
- 飞书 API 返回错误码
- 常见错误码：
  - 99991400：限流
  - 230002：机器人被移除
  - 230001：权限不足
  - 230004：群组不存在

**处理方式**：

#### 6.2.1 限流错误（99991400）

1. 显示错误提示："发送频率过快，请稍后再试"
2. 自动延迟 60 秒后重试
3. 显示倒计时："60 秒后自动重试"
4. 用户可以点击"立即重试"

#### 6.2.2 机器人被移除（230002）

1. 自动调用解绑 API
2. 显示警告提示："机器人已被移除，已自动解绑"
3. 移除状态徽章
4. 显示"同步到飞书"按钮

#### 6.2.3 权限不足（230001）

1. 显示错误提示："机器人无权限发送消息，请检查权限配置"
2. 提供"打开设置"按钮
3. 点击后打开飞书设置页面
4. 显示权限配置指引

#### 6.2.4 群组不存在（230004）

1. 自动调用解绑 API
2. 显示警告提示："飞书群组不存在，已自动解绑"
3. 移除状态徽章
4. 显示"同步到飞书"按钮

**用户体验**：
- 错误提示包含错误码
- 提供明确的操作建议
- 自动处理无法恢复的错误

### 6.3 权限不足

**场景**：
- 用户未授权飞书应用
- 飞书应用权限不足
- Token 过期

**处理方式**：
1. 显示错误提示："机器人无权限发送消息，请检查权限配置"
2. 提供"打开设置"按钮
3. 点击后打开飞书设置页面
4. 显示权限配置指引：
   - 步骤 1：打开飞书开放平台
   - 步骤 2：进入应用管理
   - 步骤 3：添加权限：im:message、im:chat
   - 步骤 4：重新授权

**用户体验**：
- 错误提示清晰
- 提供详细的操作步骤
- 引导用户解决问题

### 6.4 会话不存在

**场景**：
- 飞书群组被解散
- 机器人被移除
- 群组 ID 无效

**处理方式**：
1. 检测到错误后自动调用解绑 API
2. 显示警告提示："飞书群组不存在，已自动解绑"
3. 移除状态徽章
4. 显示"同步到飞书"按钮
5. 不需要用户操作

**用户体验**：
- 自动处理，无需用户干预
- 提示清晰，说明原因
- 允许用户重新绑定

### 6.5 绑定已存在

**场景**：
- 用户尝试绑定已绑定的会话
- 数据库中已有绑定记录

**处理方式**：
1. 检测到绑定已存在
2. 显示信息提示："该会话已绑定到飞书群组"
3. 自动打开详细信息面板
4. 显示现有绑定详情
5. 用户可以选择：
   - 查看统计
   - 暂停/恢复
   - 解绑

**用户体验**：
- 提示清晰，说明原因
- 自动显示现有绑定详情
- 提供操作选项
